# 🦠 NeuroWeave: Adversarial Cybersecurity Command Center

**NeuroWeave** is a professional-grade, interactive Adversarial Machine Learning Framework and Simulation Dashboard. It is designed to visually demonstrate the catastrophic vulnerabilities of Convolutional Neural Networks (CNNs) to gradient-based pixel manipulation, and to mathematically analyze the limits of algorithmic defense mechanisms (Adversarial Purification).

## ⚡ The Simulation

NeuroWeave allows researchers and engineers to act as both the Attacker and the Defender against a live, pre-trained ResNet-20 Neural Engine (trained on the CIFAR-10 distribution).

### 1. Organic Viral Payload Deployment (The Attack)
Upon selecting a target payload (e.g., forcing the AI to misclassify an image of a 'Cat' as a 'Dog'), the framework launches an incredibly aggressive **Projected Gradient Descent (PGD)** attack. 
* **The Mathematics:** Over 100 deep-mutation cycles, the Neural Engine's gradient pathways are mathematically calculated. The virus selectively infects pixels, crawling outward like cellular automata, and continually pushes the tensor geometry linearly towards the targeted False Reality until the Neural Engine's Softmax activation forcefully hits a mathematically perfect `100.0%`.
* **The Visualization:** The framework constructs a real-time `Mission Debrief Dashboard` containing 3D Pixel Activity Maps, Neural Collapse Timeline Graphs, and generates a dynamic GIF showing the virus spreading across the asset.
* **The Auditory Layer (Neural Scream):** To provide a visceral, multi-modal representation of network decay, NeuroWeave implements cross-modal frequency mapping. The framework translates shifting Softmax confidence values into a real-time "Neural Scream." High-confidence adversarial hallucinations generate piercing high-frequency oscillations, while mathematical uncertainty manifests as high-entropy distortion and auditory "shuddering," allowing researchers to *hear* the biological-style decay of the network logic.

### 2. Sentinel Isotropic Memory Sieve (The Defense)
The framework attempts an honest, highly advanced, zero-shot mathematical purification to salvage the asset using the `Sentinel Protocol`:
* **Non-Linear Median Scrape:** A mechanical pass that identifies and mathematically extracts severe high-frequency "salt-and-pepper" adversarial spikes.
* **Isotropic Gaussian Verification:** The continuous addition of highly controlled Gaussian noise to completely shatter the fragile, pixel-perfect floating-point alignments required by the PGD targeted attack to retain its 'Dog' hallucination.

## 🧬 Why The Sentinel Fails (The Superiority of PGD)
**The terrifying conclusion of this cybersecurity simulation is that the Defense mathematically fails by design.** 

If you successfully deploy the virus, and then activate the Sentinel Neural Shield, you will likely encounter a **"SHIELD FAILURE"** warning, where the neural network misidentifies the purified image (often as 'Airplane' or it remains compromised). 

**Why does this happen?**
This project mathematically proves the fundamental theorem of Adversarial Machine Learning: **A high-density continuous PGD attack permanently physically overwrites low-resolution source geometry.** 

Because the virus actively optimizes and destroys over 500 pixels (nearly 50% of the entire 32x32 image frame), the "real" image data is physically erased. While the Isotropic Sentinel Sieve successfully shatters the specific alignment of the *virus* (causing the engine to stop seeing "Dog"), the underlying visual structure of the original animal has been so heavily blurred and extracted through the mathematical scrubbing process that it becomes out-of-distribution noise. 

We prove that against advanced algorithmic attacks, purely reactive mathematical filters are insufficient. The damage inflicted by adversarial gradient geometry is terminal, validating the necessity for advanced Generative Diffusion purification and air-gapped cleanroom architectures in enterprise environments.

## 🚀 Execution & Implementation

### Prerequisites
* Python 3.9+
* PyTorch (Compiled for appropriate ecosystem)
* Gradio 4.0+
* Matplotlib & Pillow

### Installation
Clone the repository and install the dependencies:
```bash
git clone https://github.com/your-username/NeuroWeave.git
cd NeuroWeave
pip install -r requirements.txt
```

*(Note: PyTorch installation varies by OS and GPU availability. Ensure you install the correct wheel from `pytorch.org` first.)*

### Booting the Command Center
Launch the Web GUI directly from your terminal:
```bash
python web_app.py
```
This automatically downloads the `chenyaofo/pytorch-cifar-models` framework and initializes the localized Gradio server at `http://127.0.0.1:7860`.
